Object Mapping

Similar to OR-mapping techniques for relational databases, we provide a way to map RDFS / OWL terms to .NET objects. Using byte-code manipulation, we implement the code required for the mapping during program compilation. This results in higher performance and less runtime errors than solutions based on introspection.

Download Example 
// A class mapping to http://schema.org/Person
[RdfClass(SCHEMA.Person)]
public class Person : Resource
{
    [RdfProperty(SCHEMA.givenName)]
    public string FirstName { get; set; }

    [RdfProperty(SCHEMA.familyName)]
    public string LastName { get; set; }

    [RdfProperty(SCHEMA.birthDate)]
    public DateTime BirthDate { get; set; }
}
                  
// Create a new instance of person with a GUID
var p = Context.CreateResource<Person>();
p.Name = "John Doe";
p.Birthday = new DateTime(1985, 6, 17);
p.Commit();

LINQ

Not familiar with SPARQL? No problem. The powerful LINQ to SPARQL translator in Trinity RDF lets you query your RDF knowledge graph using LINQ. No joins attached!

Download Example 
// Create a queryable context with inferencing enabled so that the
// query also returns instances of sub-types of person.
var persons = Context.AsQueryable<Person>(true);

// Select all persons born after 1st January 2000 and
// who have a name starting with "A" or "a".
foreach (Person p in persons.Where(
  p => p.BirthDate > new DateTime(2000) &&
  p.Name.StartsWith("A", StringComparison.InvariantCultureIgnoreCase)
)
{
  Console.WriteLine(p.Name);
}

Data Binding

Many modern desktop and web application development frameworks support binding the value of .NET object properties directly to user interface controls. This allows for rapid application development and prototyping. Trinity RDF offers some time-saving attributes that let our byte-code manipulator do the heavy lifting for you.

Download Example 
// A class mapping to http://schema.org/Person
[RdfClass(SCHEMA.Person)]
public class Person : Resource
{
  // Raises the NotifyPropertyChanged-event when 
  // the property value changes.
  [RdfProperty(SCHEMA.name), NotifyPropertyChanged]
  public string Name { get; set; }
  
  // Raises the CollectionChanged-event when
  // the collection is modified.
  [RdfProperty(SCHEMA.knows)]
  public ObservableCollection<Person> KnownPersons { get; set; }
}

Data Virtualization

Knowledge graphs can become large datasets quite easily. In order to maintain application performance and user interface responsiveness, Trinity RDF makes paged queries to your datasets easy. Simply use the Take and Skip operators in your LINQ queries and you're done. Of course, SPARQL has built-in support for paging if you are creating more advanced queries.

Download Example 

var persons = Context.AsQueryable<Person>();

// Load 10 persons per page.
var pageSize = 10;

// Skip one page and load the next 10 persons.
foreach (Person p in persons.Skip(1 * pageSize).Take(pageSize))
{
  Console.WriteLine(p.Name);
}
